Auf einen Blick

Produktanbieter können mit Componento fertige Web-Komponenten bereitstellen, über die ihre Produkte in die Online-Shops von Resellern und Maklern eingebunden werden können. Die Web-Komponente kapselt die Anbindung an die API des Produktanbieters. Die Integration wird dadurch kinderleicht.

Diese Buzzwords verwendet doch inzwischen jeder... Was heißt das denn konkret? 		Das erklären wir gerne:
  • Offen: Wir setzen auf Open Source und offene Standards wie HTML 5 und Web-Components.
  • Fair: Alle Teilnehmer bei Componento begegnen sich auf Augenhöhe. Es gibt keinen großen Konzern oder keine übermächtige Plattform, die alle Regeln diktiert.
  • Transparent: Wir teilen unsere Informationen gerne mit den Partnern, Nutzern und wenn rechtlich möglich der Öffentlichkeit.
  • Demokratisch: Wir respektieren die Datensouveränität der Bürger. Niemand soll Produktangebote von uns erhalten, wenn er das nicht möchte.
Was ist der Unterschied zu einer klassischen Integration über APIs?

Auch bei Componento geschieht die eigentliche Integration in die Backend-Systeme des Produktanbieters über eine API.
Die Shops müssen aber nicht alle selber ihre Frontends erweitern und die API integrieren. Das nimmt ihnen die Web-Komponente ab. Diese muss quasi nur noch verlinkt und vom Browser geladen werden. Der Anbieter der API ist somit auch für die Integration der API (innerhalb der Web-Komponente) verantwortlich und sorgt dafür das Frontend und Backend immer aktuell und sicher bleiben.
